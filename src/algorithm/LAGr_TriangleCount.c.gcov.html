      19             : // Given a symmetric graph A with no-self edges, LAGr_TriangleCount counts the
      20             : // number of triangles in the graph.  A triangle is a clique of size three,
      21             : // that is, 3 nodes that are all pairwise connected.
      22             : 
      23             : // One of 6 methods are used, defined below where L and U are the strictly
      24             : // lower and strictly upper triangular parts of the symmetrix matrix A,
      25             : // respectively.  Each method computes the same result, ntri:
      26             : 
      27             : //  0:  default:    use the default method (currently methood 5)
      28             : //  1:  Burkhardt:  ntri = sum (sum ((A^2) .* A)) / 6
      29             : //  2:  Cohen:      ntri = sum (sum ((L * U) .* A)) / 2
      30             : //  3:  Sandia:     ntri = sum (sum ((L * L) .* L))
      31             : //  4:  Sandia2:    ntri = sum (sum ((U * U) .* U))
      32             : //  5:  SandiaDot:  ntri = sum (sum ((L * U') .* L)).  Note that L=U'.
      33             : //  6:  SandiaDot2: ntri = sum (sum ((U * L') .* U)).  Note that U=L'.
      34             : 
      35             : // A is a square symmetric matrix, of any type.  Its values are ignored.
      36             : // Results are undefined for methods 1 and 2 if self-edges exist in A.  Results
      37             : // are undefined for all methods if A is unsymmetric.
      38             : 
      39             : // The Sandia* methods all tend to be faster than the Burkhardt or Cohen
      40             : // methods.  For the largest graphs, SandiaDot tends to be fastest, except for
      41             : // the GAP-urand matrix, where the saxpy-based Sandia method (L*L.*L) is
      42             : // fastest.  For many small graphs, the saxpy-based Sandia and Sandia2 methods
      43             : // are often faster that the dot-product-based methods.
